Software Engineering Chapt 7

Software Engineering
  • Software development
  • Goal is to find principles that guide software development process leading to efficient and reliable software products.
Integrated Development Environment (ICE) Combines tools to develop software into an integrated (single) package.
Association for Computing Machinery (ACM)
  • Founded in 1947.
  • International and educational organization dedicated to the advancement of the arts, sciences, and applications of Information Technology.
Purpose of ACM Brings together computing educators, researchers and professionals to inspire dialogue, share resources, and address the challenges of the field.
Benefits of an ACM membership.
  • Leadership opportunities on projects.
  • Lifelong learning programs.
  • Speacial intrest groups with local chapters.
Student ACM membership
  • Option to subscribe to full ACM digital library.
  • Email forwarding address with spam filtering.
  • Online subscriptions to numerous magazines in the related field.
  • Quarterly newsletter.
Institute of Electrical and Electronics Engineers (IEEE)
  • Formed in 1963.
  • Merged American Institute of Electrical Engineers and Institute of Radio Engineers.
  • Involved with the developement of standards.
Purpose of IEEE To foster technological innovation and excellence for the benefit of the community.
Benefits of IEEE membership
  • Keeping technically current with subscriptions to magazines.
  • Career resources and recognition.
  • Professional networking.
  • Continuing education.
  • Discounts.
  • Humanitarian programs.
Student IEEE member benefits.
  • Different levels of membership based upon graduation date.
  • First elevation: Given a ballot to vote in IEEE September elections.
  • Second elevation: Graduates recieve 50% off the membership dues (once in a lifetime discount).
Software Life Cycle: Development Phase
Requirements Analysis Developing a set of requirements that identify features and services the finished software product should have.
Design Creates a plan for the construction of the proposed system
Implementation Actual writing of a program
Testing Makes sure software is compatable with required specifications.
Modular Implementation Breaking a system into parts, modules, for testing purposes.
Unified Modeling Language (UML) Collection of tools developed with an object-oriented paradigm.
Use Case
  • Computer aided software engineering.
  • Application of computer technology for the software development process.
  • Simplifying the development process.
Use Case Diagram Attempts to capture image of proposed system from users point of view.
Data Relationships
One to One How one class of data is related to one other class of data.
One to Many One class of data is related to many classes of data.
Many to Many Multiple classes of data are related to multiple classes of data
Design Pattern Predeveloped model for solving a recurring problem in software design.
Software Quality Assurance (SQA) Groups that are in charge of overseeing and enforcing quality control shystems adopted by the organization.
Software Testing
Glass Box Testing Situation where the tester is aware of the interior structure of the software and uses that knowledge when designing the test.
Black Box Testing Tests that do not rely on knowledge of the softwares interior composition.
Beta Testing A Segment of the preliminary version is given to intended audience.
Alpha Testing Testing done at the developers site; similar to beta testing.
Documentation
User Documentation Describes features of software and how to operate it.
System Documentation Describes a softwares internal composition for future maintenance.
Technical Documentation Describes the instillation and service of the software.
Intelectual Property Law Gives a level of ownership to software developers.
Software License
  • Developer rights are formally expressed.
  • Legal agreement between owner and the user of a software product.
  • Grants user certain permissions to use product without transferring rights.
Software Disclaimer Protects owner from liability.
The best defense against software liability claims:
  • Apply sound software engineering principles during the developement of software.
  • Use a level of care compatible with the sodtwares application.
  • Produce and maintain records that validate those endeavors.
