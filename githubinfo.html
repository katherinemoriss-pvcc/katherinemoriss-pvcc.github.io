All About Github

Term Definition
Repository Virtual storage space for files, codes and their revision history.
Commit Snapshot of project changes saved to a local repository.
Fork Process of making a copy of a repository storing it in your own account.
Push Used to upload local repository content to remote
Pull Request Merge a set of changes from one branch to another.
Users can review and confer proposed set of changes into main code base.
Workflows Configurable automated process that runs on or more jobs.
Define by YAML file in repository.
Issues Items created in a repository to plan, discuss and track work.
Raw Button Allows users to view or copy raw content of a file without styling or HTML formatting.
Blame Button Git blame command used to examine contents of a file in depth.

What is a Version Control System (VCS)

  1. Tracks th history of changes when people or teams collaborate on projects.
  2. Earlier versions of project can be recovered.
  3. Developers can review the history of a project to see what changes have been made.

What is Git?

  1. A free open source distributed version control system.
  2. Designed to handle small to large projects with efficiency and speed.

What is Github?

  1. Cloud based platform.
  2. User can work with others to develop codes or make changes an existing projects.
  3. Can share files, track and manage changes to codes.
  4. Other people can review code and make changes over time.
  5. Suggestions can be made on projects within Github.

What is the difference between Git and Github?

Git:

  1. Installed on a developers computer
  2. Changes made within files are tracked
  3. Multipe people can work on the same files at the same time
    by using pushing and pull.
  4. Cannot see other peoples work in real time.

Github

  1. Cloud based.
  2. Development is built on Git.
  3. Some security features include: authentication, access controls,
    and code scanning tools.
  4. Users can share and collaborate on files.
  5. Makes Git easier to use.
  6. Web interface to perform tasks.

Who started Github and how was it started?

  1. Idea began with a Finnish Software Engineer named Linus Torvalds in 2005.
  2. Developed in October of 2007
  3. Was made available as a beta release for a few months.
  4. Developed by: Tom-Preston Werner, Chris Wanstrath,
    P.J. Hyett and Scott Chacon.
  5. Launched April 2008.
