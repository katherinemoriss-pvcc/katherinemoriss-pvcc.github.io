Software Testing & Verification

Kinds of Tests

Term Definition
Acceptance Testing Verifying whether the whole system works as intended.
Integration Testing Ensuring that software components or functions operate.
Unit Testing Validating that each software unit runs as expected.
Funtional Testing Checking functions by creating business scenarios based on funtional requirements.
Performance Testing Testing how the software runs under different workloads.Ex: Load Testing
Regression Testing Checking whether new features break or degrade functionality. Ex: Sanity Testing
Stress Testing Testing to see how much strain the sytem can take before it fails; type of non-functional testing.
Usability Testing Validating how well a customer can use a system or web application to complete a task.

Phases of Computer Program Development

  1. Phase 1: Understand the problem.
  2. Phase 2: Design an algorithm to solve the problem.
  3. Phase 3: Write the program code in appropriate language based on the algorithm.
  4. Phase 4: Enter the program and save.
  5. Phase 5: Execute the program.
  6. Phase 6: Test the program, find errors, debug then retest.
  7. Phase 7: Document and evaluate the problem.

Programming Structures

  2. Decision Structures: if/then/else

Classing Program Goals:

    Efficiency
  • Program takes minimal time and resources.
    Correctness
  • Programs run in expected ways and works for all expected conditions.

    • Testing is done to detect:

    • Architectural flaws
    • Poor design decisions
    • Invalid or incorrect functionality
    • Security Vulnerability
    • Social ability issues

    Virtualization

    • Simulates services and systems that are missing or not yet completed.
    • Enables teams to reduce dependencies and test sooner.
    • Can reuse, deploy or change configurations, testing different scenarios without modifying the original environment.

    Defect or "bug tracking"

    • Allows teams to track defects and measure their scope and inpact, uncover related issues

    Metrics and Reporting

    • Reporting and analytics enable team members to share status, goals and test results.
    • Advance tools integrate project metrics and present results in dashboard.
    • Quickly see overall health of the project and can monitor.
